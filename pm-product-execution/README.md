# pm-product-execution

Day-to-day product management execution skills from PRD to release.

## Skills (7)

- **prd-writer** — Write a comprehensive 8-section Product Requirements Document.
- **okr-planner** — Plan team-level OKRs cascaded from company goals with KR quality checks.
- **roadmap-builder** — Build outcome-based roadmaps in Now/Next/Later, outcome, or theme formats.
- **user-stories** — Write user stories and job stories with Given/When/Then acceptance criteria.
- **stakeholder-management** — Map stakeholders on a Power/Interest grid with RACI matrix and communication plan.
- **sprint-management** — Plan sprints, run retrospectives, write release notes, and run pre-mortems.
- **metrics-framework** — Define North Star Metric, input metrics, and guardrail metrics.

## Commands (8)

- `/pm-product-execution:write-prd` — Write a complete PRD from a feature idea or problem statement.
- `/pm-product-execution:plan-okrs` — Write team-level OKRs cascaded from company goals.
- `/pm-product-execution:roadmap` — Build or transform a product roadmap.
- `/pm-product-execution:write-stories` — Break a feature into sprint-ready user stories.
- `/pm-product-execution:sprint-plan` — Plan a sprint with capacity estimation and sprint goal.
- `/pm-product-execution:retro` — Run a sprint retrospective or write release notes.
- `/pm-product-execution:define-metrics` — Define a North Star Metric and supporting metrics framework.
- `/pm-product-execution:stakeholder-map` — Map stakeholders and design a communication plan.
