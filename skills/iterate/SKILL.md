# Iterate

This skill routes to **product-iteration**, which provides a complete post-launch iteration framework using Build-Measure-Learn, PDCA/Deming cycle, and hypothesis-driven development.

**When to use this:**
- A feature shipped and performance is below target — you need a structured improvement plan
- You have qualitative feedback and usage data and want to turn them into prioritized changes
- Running a continuous improvement cadence for a mature product area
- Preventing the team from declaring "done" after first launch and moving on

**Quick iteration mode guide:**
- Have data and feedback, need a plan → Full iteration cycle: diagnose → hypothesize → design experiment → prioritize
- Performance gap is clear, cause unknown → Diagnose first: are metrics below target because of awareness, adoption, or behavior?
- Have a backlog of improvement ideas → Apply hypothesis-driven prioritization before building any of them
- Running weekly iteration sprints → Use PDCA cadence as the operating rhythm

**What product-iteration covers:**
- Build-Measure-Learn loop (Lean Startup): structure for each iteration cycle
- PDCA/Deming cycle: Plan-Do-Check-Act applied to product iteration
- Hypothesis-driven development: every change has a stated hypothesis, metric, and pass/fail threshold before development starts
- Root cause analysis: 5 Whys, fishbone diagram applied to product performance gaps
- Iteration vs. pivot decision: when data says "improve this" vs. "change direction entirely"
- Feedback synthesis: converting qualitative VoC data + quantitative usage data into prioritized iteration areas
- Jobs-to-be-Done iteration lens: is the feature helping users complete the job more effectively?
- Iteration backlog structure: hypothesis, expected metric move, effort estimate, test type
- Anti-patterns: shipping iterations without measurement, iterating without a hypothesis, optimizing local metrics that don't move the North Star

**The skill will:**
1. Ask for feature context, performance data, and any user feedback
2. Optionally load voice-of-customer skill first if raw feedback needs synthesis
3. Diagnose whether the gap is awareness, adoption, or behavior-level
4. Generate 5+ iteration hypotheses, each with a stated metric and threshold
5. Prioritize the iteration backlog by expected learning value
6. Design the first iteration cycle with success criteria
7. Build the measurement plan for each change
