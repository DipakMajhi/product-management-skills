# Interview Prep

This skill routes to **interview-prep-coach**, which provides expert PM interview coaching with company-specific patterns, frameworks, and scored critique.

**When to use this:**
- Preparing for a specific PM interview (product design, metrics, strategy, behavioral)
- Practicing your answer to a specific question and getting critique
- Learning which frameworks to use for which interview question types
- Understanding what top-tier companies (Google, Meta, Amazon, Stripe) look for at each level
- Running mock interviews to sharpen delivery before the real thing

**Quick guide by question type:**
- "Design a product for X" → Product design framework: user → segment → goals → pain points → solutions → metrics
- "How would you improve [product]?" → Improvement framework: same as above, anchored in existing product
- "You notice metric X dropped. What do you do?" → Metrics investigation: clarify → hypothesize → segment → root cause → fix
- "How would you prioritize A, B, C?" → Prioritization framework: align on goal → framework selection → score → recommend
- "Tell me about a time you…" → Behavioral (STAR): Situation, Task, Action, Result — with quantified outcomes
- "How big is the market for X?" → Estimation: Fermi decomposition with explicit assumptions

**What interview-prep-coach covers:**
- CIRCLES method (Lewis Lin) for product design questions
- Metrics framework: clarify, confirm, hypothesize, segment, diagnose, prioritize, validate
- Estimation methodology: Fermi decomposition, population segmentation, unit-based estimation
- Behavioral interview preparation: STAR structure, most common behavioral themes, anti-patterns
- Company-specific patterns: Google (user empathy + data), Meta (growth + experimentation), Amazon (Customer Obsession + Working Backwards), Stripe (systems thinking + rigor)
- Level-specific expectations: what separates a PM answer from a Senior PM answer from a Staff PM answer
- Answer critique rubric: structure, depth, user empathy, data use, strategic thinking, communication
- Common PM interview mistakes: jumping to solutions, no metrics defined, generic answers, too tactical
- Follow-up question handling: how to engage with "why did you choose that?" and "what would you do differently?"
- How to open PM interview answers: the first 30 seconds set the scoring trajectory

**The skill will:**
1. Ask what type of interview you're preparing for and the company/level if known
2. Present the relevant framework for that question type
3. Walk through a sample question using the framework
4. Critique your answer or provide a model answer to compare
5. Score the answer against the rubric
6. Identify the specific element to improve (structure, depth, data, user empathy)
7. Repeat until the answer is at the target level
