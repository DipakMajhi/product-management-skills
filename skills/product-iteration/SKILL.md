# Product Iteration Framework

Iteration is how products get from "launched" to "great." Most launched features need 2-3 iteration cycles before they work well. A structured iteration process turns feedback into improvements systematically.

## The Improvement Interview Framework

Apply this skill to: $ARGUMENTS


Used in PM interviews: "How would you improve [existing product]?"

**Step 1: Clarify the goal**
What does "improve" mean? Improve for whom? Improve on what metric?

**Step 2: Define user segments**
Who are the different types of users? Which segment should we focus on improving for?

**Step 3: Identify pain points**
For the target segment, what are their biggest frustrations, unmet needs, or moments of confusion?

**Step 4: Prioritize pain points**
Which pain points are most common, most severe, and most aligned with the business goal?

**Step 5: Ideate solutions**
For the top pain point, generate 3-5 solution ideas of different types (quick fix, medium investment, big bet).

**Step 6: Prioritize solutions**
Which solution has the best impact/effort/confidence ratio?

**Step 7: Define success metrics**
How will you know if the improvement worked?

## The Post-Launch Iteration Cycle

After a feature ships:

**Week 1-2: Monitor**
- Is the feature being discovered and used?
- Are there any critical bugs or error rates spiking?
- What does early qualitative feedback say?

**Week 3-4: Diagnose**
- Compare actual usage to predicted usage
- If below expectations: activation problem or awareness problem?
- If at expectations: look at depth of use and satisfaction

**Month 2: First Iteration**
- Take the top 2-3 learnings from launch data and feedback
- Define and ship the highest-impact low-effort improvements
- Retest with a targeted user group if possible

**Month 3+: Depth Iteration**
- Build deeper features for power users who have adopted the core
- Address the pain points that are causing churn from the feature
