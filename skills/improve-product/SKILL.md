# Improve Product

This skill routes to **product-iteration**, which provides the complete PM Improvement Interview Framework and structured product iteration methodology.

**When to use this:**
- Preparing for PM interview questions: "How would you improve [product]?"
- Building a structured case for improving a specific product area
- Thinking through product improvements beyond surface-level feature ideas
- Practicing how to structure improvement thinking with rigor, not just opinions

**Quick interview prep guide:**
- Google/Meta/Amazon PM interview → Walk the full 7-step framework explicitly; interviewers score on structure
- Early-stage startup context → Focus on discovery-led iteration: identify the highest-value opportunity first
- Product critique exercise → Use the framework to avoid jumping to solutions before defining users and goals
- Prioritization follow-up → After identifying improvements, use RICE or MoSCoW to rank them

**What product-iteration covers:**
- PM Improvement Interview Framework (7 steps): clarify scope → identify user segments → pick a segment → map goals and pain points → prioritize pain points → generate solutions → prioritize solutions → define success metrics
- How to pick the right user segment: volume, intensity of pain, strategic importance
- Jobs-to-be-Done (JTBD) application: framing improvements as job-completion improvements, not feature additions
- Divergent vs. convergent thinking: generating many ideas before narrowing (avoids anchoring on the obvious)
- Solution evaluation: feasibility, impact, fit with product direction
- Success metric definition: which metric would move if this improvement worked?
- Build-Measure-Learn loop: how to iterate in short cycles after shipping
- PDCA/Deming cycle: Plan-Do-Check-Act for continuous improvement discipline
- Hypothesis-driven development: stating what you believe before building
- Product improvement anti-patterns: copy competitors, build the most-requested feature, optimize the wrong metric

**The skill will:**
1. Ask which product and which user segment to focus on (or help narrow if you're unsure)
2. Walk through the 7-step framework explicitly, showing the reasoning at each step
3. Identify and prioritize 3-5 user pain points with supporting logic
4. Generate 5+ solution ideas, including non-obvious ones
5. Prioritize solutions by impact and feasibility
6. Define success metrics for the top 1-2 solutions
7. Deliver an interview-ready, structured answer
