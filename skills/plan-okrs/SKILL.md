# Plan OKRs

This skill routes to **metrics-and-okrs** Mode C (OKRs), which provides a complete OKR writing and review framework using John Doerr's methodology.

**When to use this:**
- Writing OKRs for a new quarter or half-year
- Reviewing and improving existing OKR drafts before finalizing
- Cascading from a company or department OKR to a product team OKR
- Helping a team distinguish good Key Results from activity-based tasks
- Answering "how would you write OKRs for this team?" in PM interviews

**Quick guide by situation:**
- Writing from scratch → Start with the Objective (aspiration) before any Key Results
- Cascading from company OKRs → Identify your team's specific contribution, not a copy of the parent OKR
- Reviewing existing KRs → Apply the quality test: measurable, from-X-to-Y, outcome not task, has an owner
- Too many OKRs → Ruthlessly cut to max 3 Objectives, max 4 KRs per Objective
- Ambition calibration → Target 70% achievement; consistent 100% means your KRs aren't ambitious enough

**What metrics-and-okrs Mode C covers:**
- John Doerr's OKR methodology from Measure What Matters
- Objective criteria: qualitative, inspirational, time-bounded, directional, not a metric
- Key Result criteria: measurable, from-X-to-Y format, outcome not task, clear owner, no more than 4 per Objective
- Common OKR mistakes: task KRs, vanity KRs, activity KRs, too many KRs, objectives that are actually projects
- Cascading framework: company → department → team → individual
- OKR ambition calibration: the 70% target and what happens when you always hit 100%
- CFR framework (John Doerr): Conversations, Feedback, Recognition — the operating cadence around OKRs
- Weekly check-in format: confidence score + blocker + update
- OKR grading at end of cycle: 0.7 is success, 1.0 means it was too easy, 0.3 means something went wrong
- OKR templates: team OKR format, individual OKR format, cascade alignment table

**The skill will:**
1. Ask about the planning period and any company/department OKRs to cascade from
2. Help define the Objective (aspiration, not a metric)
3. Write 3-4 Key Results using the from-X-to-Y format with owners
4. Review each KR against the quality test and flag task-based or vanity KRs
5. Check ambition calibration — are these achievable at 70%, meaningful at 100%?
6. Set up the weekly check-in cadence
7. Deliver formatted OKR output ready for a planning doc
