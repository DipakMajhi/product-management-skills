# Roadmap

This skill routes to **roadmap-builder**, which provides five roadmap formats with audience-specific guidance and a feature-to-outcome transformation tool.

**When to use this:**
- Building a roadmap from scratch from a feature backlog or OKRs
- Transforming a date-driven feature list into an outcome-based roadmap
- Preparing roadmaps for different audiences: engineering, board, sales, customers
- Deciding between roadmap formats (Now/Next/Later vs. theme-based vs. GIST)
- Answering "how would you build a roadmap for this team?" in PM interviews

**Quick format guide by audience:**
- Engineering / cross-functional team → Now/Next/Later (Janna Bastow): confidence levels, no false dates
- Board / investors → Theme-based: strategic story, not feature list
- Sales team → Outcome-based: tied to customer pain points and commitments
- Product-led team running OKRs → Outcome-driven (Josh Seiden): outcomes as rows, initiatives as children
- Teams with high delivery uncertainty → GIST (Itamar Gilad): Goals → Ideas → Step-Projects → Tasks

**What roadmap-builder covers:**
- Now/Next/Later format: confidence levels, time horizon, commitment vs. direction distinction
- Outcome-driven roadmap: grouping features by the outcome they drive, not by feature category
- Theme-based roadmap: strategic narrative format for board and leadership communication
- Opportunity-driven roadmap (Teresa Torres): OST as input, opportunities as rows
- GIST framework (Itamar Gilad): goals → idea bank → quarterly step-projects → sprint tasks
- Feature-to-outcome transformation: taking a raw feature list and regrouping by customer outcome
- Confidence levels and commitment spectrum: "we will" vs. "we plan to" vs. "we're exploring"
- Dependency mapping: identifying cross-team and cross-feature sequencing constraints
- Roadmap review cadence: weekly vs. monthly vs. quarterly updates, version control
- Audience-specific adaptation: same roadmap, different abstraction levels
- Anti-patterns: date-driven roadmaps that become promises, too much detail too far out, roadmaps divorced from strategy

**The skill will:**
1. Ask about the audience, time horizon, and current state (feature list, backlog, or rough ideas)
2. Recommend the right format for the context
3. Group features or initiatives by outcome if given a feature list
4. Apply confidence levels rather than fixed dates
5. Map dependencies that affect sequencing
6. Produce the formatted roadmap document
7. Offer to adapt it for a second audience if needed
