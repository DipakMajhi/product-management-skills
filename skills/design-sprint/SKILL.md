# Design Sprint

This skill routes to **design-thinking**, which provides complete facilitation guides for full design sprints, compressed sessions, and individual design thinking stages.

**When to use this:**
- Running a 4 or 5-day Google Design Sprint with your team
- Facilitating a compressed half-day or full-day design session
- Applying design thinking to a single, well-scoped problem
- Generating How Might We (HMW) questions to reframe a problem
- Running structured ideation (Crazy 8s, SCAMPER, analogous inspiration)
- Answering "how would you run a design sprint?" in PM interviews

**Quick format guide by time available:**
- 5 days → Full Google Design Sprint (Jake Knapp): map, sketch, decide, prototype, test
- 4 days → Design Sprint 2.0 (AJ&Smart): combined day 1+2, same outcomes
- 1 day → Double Diamond compressed: problem framing in the morning, ideation in the afternoon
- 30-60 min → Lightning Decision Jam: fastest path from messy problem to prioritized actions
- Ongoing → IDEO 5-stage iterative process embedded in the product development cycle

**What design-thinking covers:**
- IDEO 5-Stage Model: Empathize → Define → Ideate → Prototype → Test
- Double Diamond (British Design Council): Discover + Define (problem diamond) + Develop + Deliver (solution diamond)
- Google Design Sprint: full 5-day and 4-day formats with daily agendas
- Lightning Decision Jam (AJ&Smart): 7-step facilitation for rapid alignment
- How Might We (HMW) reframing: formula, examples, anti-patterns
- 8 ideation techniques: Crazy 8s, SCAMPER, Brainwriting 6-3-5, Worst Possible Idea, Round-Robin, Bodystorming, Analogous Inspiration, Constraint Ideation
- Prototyping spectrum: paper sketch → storyboard → clickable mockup → role play → Wizard of Oz
- 6 facilitation techniques: Dot Voting, Rose/Thorn/Bud, Affinity Clustering, Silent Critique, Gallery Walk, Decider Vote
- How to pick a sprint target: long-term goal → sprint questions → target area selection
- Anti-patterns: jumping to solutions, HIPPo influence, design by committee, untested prototypes

**The skill will:**
1. Ask about the problem, user, available time, and team composition
2. Recommend the right sprint format for your time and context
3. Provide a full agenda with facilitation instructions for each stage
4. Generate HMW statements from the problem description
5. Run or describe structured ideation for the target
6. Define the prototype plan and user test structure
7. Help synthesize insights after user testing
